## 1. 设计原则

### 1.1 核心设计理念

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "内容为王，操作极简"                                           │
│                                                                 │
│   • 用户打开即看到内容，不是功能入口                              │
│   • 每个操作不超过 2 步                                          │
│   • 减少决策，突出推荐                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 Apple 设计原则

| 原则 | 说明 |
|------|------|
| **清晰性** | 文字清晰易读，图标精准明确，功能一目了然 |
| **遵从性** | 界面退居幕后，内容成为主角 |
| **深度感** | 分层视觉层次，合理的动效反馈 |

### 1.3 设计价值观

| 场景 | 设计目标 |
|------|----------|
| 通勤场景 | 单手操作，快速浏览 |
| 候机场景 | 离线友好，深度沉浸 |
| 休息场景 | 视觉舒适，无压力阅读 |

## 5. 交互规范

### 5.1 手势与操作

| 操作 | 触发条件 | 效果 |
|------|----------|------|
| 点击卡片 | 点击卡片任意区域 | 进入详情页 |
| 点击收藏 | 点击[收藏]按钮 | 卡片添加收藏标记，按钮变已收藏 |
| 点击跳过 | 点击[跳过]按钮 | 卡片滑出，显示下一篇 |
| 下拉刷新 | 首页下拉 | 刷新今日论文 |
| 上拉加载 | 滚动到底部 | 加载更多论文 |

### 5.2 动效规范

| 场景 | 动效 | 时长 |
|------|------|------|
| 卡片进入 | 从下方淡入滑动 | 300ms |
| 卡片跳过 | 向左滑出淡出 | 250ms |
| Tab 切换 | 内容淡入淡出 | 200ms |
| 页面进入 | 从右滑入 | 300ms |
| 页面返回 | 向右滑出 | 250ms |
| 按钮点击 | 缩放 0.95 → 1 | 100ms |

### 5.3 触觉反馈

```javascript
// 按钮点击
button.onTap = () => wx.vibrateShort({ type: 'light' });

// 收藏成功
bookmark.onSuccess = () => wx.vibrateShort({ type: 'success' });

// 操作失败
onError = () => wx.vibrateShort({ type: 'error' });
```
