# ADR-002: 微信小程序技术选型

## 状态

已采纳（2026-03-27 更新：从原生框架改为 uni-app）

## 背景

PaperLens V1.0 以微信小程序为主力客户端。小程序开发有多种技术选型：

1. **原生框架**：WXML/WXSS/JS
2. **跨端框架**：Taro、uni-app、Remax

## 决策

选用 **uni-app（Vue 3）**。

## 变更历史

| 时间 | 决策 | 原因 |
|------|------|------|
| 2026-03-26 | 微信小程序原生框架 | 审核友好、包体小、性能优 |
| 2026-03-27 | **uni-app（Vue 3）** | 团队熟悉 Vue 生态、社区成熟、后续可扩展多端 |

## 理由

### 1. 团队熟悉 Vue 生态

- 团队成员对 Vue 语法熟悉，开发效率高
- 可复用 Vue 组件和开发经验
- 学习成本低

### 2. 社区成熟，文档丰富

- uni-app 社区活跃，问题易解决
- 插件市场丰富，可快速集成功能
- 官方文档完善

### 3. 多端扩展能力

- 一套代码可编译到微信、支付宝、抖音等多端小程序
- 未来可扩展到 H5、App
- 为后续多端支持预留空间

### 4. 审核风险可控

- uni-app 编译后为原生代码，审核通过率高
- 选择稳定版本，避免新特性风险
- 已有大量 uni-app 小程序成功上线案例

## 替代方案

| 方案 | 优点 | 缺点 | 结论 |
|---|---|---|---|
| 原生框架 | 审核友好、性能优、包体小 | 仅限微信，不跨端 | ❌ 团队 Vue 熟悉度优先 |
| Taro | 多端一套代码、React 语法 | 包体大、团队不熟悉 React | ❌ 不适合 |
| **uni-app** | Vue 语法、多端、社区成熟 | 包体略大（~100KB） | ✅ 采纳 |
| Remax | React 语法 | 社区较小 | ❌ 不适合 |

## 技术细节

### 项目结构

```
paperlens-uniapp/
├── api/           # API 封装层
├── stores/        # 状态管理
├── pages/         # 页面
│   ├── index/     # 论文列表
│   ├── detail/    # 论文详情
│   └── settings/  # 设置
└── static/        # 静态资源
```

### 核心依赖

- uni-app（Vue 3 版本）
- 无额外状态管理库（使用简单的 reactive store）

### 编译目标

- 主要：微信小程序
- 未来：支付宝小程序、H5、App

## 后果

- **正面**：
  - 开发效率高，Vue 语法熟悉
  - 社区支持好，问题易解决
  - 具备多端扩展能力

- **负面**：
  - 包体比原生略大（约 100KB 运行时）
  - 编译链增加，调试复杂度略高

- **缓解**：
  - 使用稳定的 uni-app 版本
  - 关注编译产物体积，按需引入组件
