## 1. 产品定位

### 1.1 核心理念

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "让学术阅读像刷短视频一样轻松"                                   │
│                                                                 │
│   • 3秒内判断论文是否感兴趣                                        │
│   • 30秒内获取 AI 精华摘要                                        │
│   • 1分钟内完成深度对话                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 目标用户

| 用户群体 | 核心场景 | 核心诉求 |
|----------|----------|----------|
| 学术研究人员 | 通勤/等待时间 | 追踪顶级期刊/会议动态，不遗漏重要论文 |
| 算法工程师 | 碎片化阅读 | 快速了解可落地的技术细节 |
| 在校学生 | 日常学习 | 克服语言障碍，快速理解核心思想 |
| 科研爱好者 | 兴趣探索 | 发现有趣的前沿研究 |

### 1.3 差异化定位

| 维度 | 传统方式 | PaperLens |
|------|----------|-----------|
| 浏览方式 | 列表翻页 | 卡片滑动 |
| 内容消费 | 阅读全文 | AI 摘要优先 |
| 交互深度 | 浅层浏览 | AI 对话深入 |
| 数据来源 | 单一平台 | 多源聚合 |
| 个性化 | 无 | 基于行为的智能推荐 |

**交互手势定义**：

| 手势 | 动作 | 数据收集 | 后续影响 |
|------|------|----------|----------|
| 左划 | 感兴趣 | +1 正向反馈 | 提升同类论文推荐权重 |
| 右划 | 不感兴趣 | +1 负向反馈 | 降低同类论文推荐权重 |
| 上划 | 查看详情 | +1 深度阅读 | 记录阅读时长 |
| 下划 | 分享 | +1 分享行为 | 记录分享偏好 |
| 双击 | 快速收藏 | +1 收藏 | 加入收藏夹 |
| 长按 | 快捷菜单 | - | 复制链接/笔记/标签 |

#### 3.2.2 对话详情页

**对话界面**：

```
┌─────────────────────────────────────────────────────────────────┐
│  ← Transformer 架构原理                                    ⋮   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │                                                          │  │
│  │  用户  10:30                                            │  │
│  │  ┌────────────────────────────────────────────────────┐ │  │
│  │  │  请解释一下 Transformer 的自注意力机制               │ │  │
│  │  └────────────────────────────────────────────────────┘ │  │
│  │                                                          │  │
│  │  AI 助手  10:30                                         │  │
│  │  ┌────────────────────────────────────────────────────┐ │  │
│  │  │  自注意力机制是 Transformer 的核心创新，它允许      │ │  │
│  │  │  模型在处理序列时，每个位置都能直接关注到序列      │ │  │
│  │  │  中的所有其他位置。                                 │ │  │
│  │  │                                                     │ │  │
│  │  │  核心公式：                                         │ │  │
│  │  │  Attention(Q,K,V) = softmax(QK^T/√d)V             │ │  │
│  │  │                                                     │ │  │
│  │  │  简单来说：                                         │ │  │
│  │  │  • Q (Query): 查询向量，"我想找什么"               │ │  │
│  │  │  • K (Key): 键向量，"我有什么特征"                 │ │  │
```

## 7. 技术实现要点

### 7.1 滑动交互实现

```javascript
// 滑动卡片组件核心逻辑
Component({
  data: {
    startX: 0,
    startY: 0,
    currentX: 0,
    currentY: 0,
    isDragging: false,
    swipeDirection: null
  },

  onTouchStart(e) {
    this.setData({
      startX: e.touches[0].clientX,
      startY: e.touches[0].clientY,
      isDragging: true
    });
  },

  onTouchMove(e) {
    if (!this.data.isDragging) return;

    const deltaX = e.touches[0].clientX - this.data.startX;
    const deltaY = e.touches[0].clientY - this.data.startY;

    // 判断滑动方向
    if (Math.abs(deltaX) > Math.abs(deltaY)) {
      // 水平滑动
      this.setData({
        currentX: deltaX,
        swipeDirection: deltaX > 0 ? 'right' : 'left'
      });
    } else {
      // 垂直滑动
      this.setData({
        currentY: deltaY,
        swipeDirection: deltaY > 0 ? 'down' : 'up'
      });
    }
  },

  onTouchEnd() {
    const { currentX, currentY } = this.data;
    const threshold = 100; // 滑动阈值

    if (Math.abs(currentX) > threshold) {
      // 触发水平滑动
      if (currentX > 0) {
        this.triggerEvent('swiperight');
      } else {
        this.triggerEvent('swipeleft');
      }
    } else if (Math.abs(currentY) > threshold) {
      // 触发垂直滑动
      if (currentY > 0) {
        this.triggerEvent('swipedown');
      } else {
        this.triggerEvent('swipeup');
      }
    }

    // 重置状态
    this.setData({
      startX: 0,
      startY: 0,
      currentX: 0,
      currentY: 0,
      isDragging: false,
      swipeDirection: null
    });
  }
});
```

### 7.2 数据存储设计

#### 本地存储 (Storage)

```javascript
// 用户偏好缓存
const PREFERENCE_KEY = 'user_preferences';

// 保存偏好
function savePreference(preference) {
  wx.setStorageSync(PREFERENCE_KEY, preference);
}

// 读取偏好
function getPreference() {
  return wx.getStorageSync(PREFERENCE_KEY) || defaultPreference;
}

// 更新单个偏好
function updatePreference(key, value) {
  const preference = getPreference();
  preference[key] = value;
  savePreference(preference);
}
```

#### 服务端数据同步

```javascript
// 同步用户偏好到服务端
async function syncPreferences() {
  const localPreference = getPreference();

  try {
    await request('/user/preferences', {
      method: 'PUT',
      data: localPreference
    });
  } catch (e) {
    // 离线时保存在本地，下次同步
    console.error('同步偏好失败', e);
  }
}
```
