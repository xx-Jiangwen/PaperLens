## 1. 产品定位

### 1.1 核心定位

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   "每日论文简报，碎片时间高效筛选"                                 │
│                                                                 │
│   小程序 = 筛选器（碎片化浏览）                                    │
│   PC端 = 研究站（深度阅读）                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 目标用户

| 用户群体 | 核心场景 | 核心诉求 |
|----------|----------|----------|
| 学术研究人员 | 通勤/等待时间 | 追踪顶级期刊/会议动态，不遗漏重要论文 |
| 算法工程师 | 碎片化阅读 | 快速了解可落地的技术细节 |
| 在校学生 | 日常学习 | 克服语言障碍，快速理解核心思想 |

### 1.3 核心差异化

| 维度 | 传统方式 | PaperLens |
|------|----------|-----------|
| 心理模型 | 主动搜索 | 被动接收精选 |
| 浏览方式 | 列表翻页 | 每日简报卡片流 |
| 决策成本 | 需要深度阅读才能判断 | 看摘要即可筛选 |
| 个性化 | 无 | 基于兴趣的智能推荐 |

|------|------|
| 原文摘要 | 优先展示，可展开/收起 |

| 轻量问答 | 仅围绕当前论文，不保留历史 |

| 分享 | 微信转发给好友/群聊 |

## 4. 交互设计

### 4.1 用户反馈机制

| 操作 | 数据收集 | 后续影响 |
|------|----------|----------|
| 点击 [收藏] | +1 正向反馈 | 提升同类论文推荐权重 |
| 点击 [跳过] | +1 负向反馈 | 降低同类论文推荐权重 |
| 点击详情页 | +1 深度阅读 | 记录阅读时长 |
| 分享 | +1 分享行为 | 记录分享偏好 |
